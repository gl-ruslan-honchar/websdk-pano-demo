Constructor

# new Player(container, optionsopt)

WebSKD Player

Base Pixellot WebSDK Player player class from which you can create video player.

Parameters:
Name Type Attributes Description
container HTMLElement

player container
options object <optional>

core WebSDK Player options
autoplay string | boolean <optional>

autoplay option could be one from bellow values

  1. a boolean value of false: won't autoplay

  2. a boolean value of true: will use browsers autoplay

  3. a string value of 'muted': will mute the video element and then manually call play() on loadstart

  4. a string value of 'play': will call play() on loadstart, similar to browsers autoplay

  5. a string value of 'any': will call play() on loadstart, and if the promise is rejected it will mute the video element then call play()

Basically options are transparent to the VideoJS options which are 'inside of the box'. But we don't support it fully, so recommended to use this only as boolean value.
pano Player~PanoOptions <optional>

set of options for configuration pano mode functionality.

# registerManager(managerData)

Register manager on player.

Basically manager represents just a property on player instance and behaves just like a plugin.

The main difference between 'manager' and 'plugin' is that manager attaches to a single property on player, but plugin is extending player with it's 'exposedProps'

Parameters:
Name Type Description
managerData Player~PlayerManagerOption

data of player manager with it's class and additional params
Example
class TestManager {
    constructor (player, options) {
        this.testProp = options.foo
    }

    test () {
        console.log(this.testProp)
    }
}

const manager =  { name: 'testManager', model: TestManager, options: { foo: 123 } }

player.registerManager(manager)

// usage
player.testManager.test() // 123

# setSource(url, preRollopt, midRollsopt, modeopt)

Set video source(s) to the player with ads

Parameters:
Name Type Attributes Default Description
url string | object

the string or object of source video URL if url is string - will be set to hd mode by default if url is an object than expects hd/pano props to be set accordingly
hd string <optional>

source to video in hd mode
pano string <optional>

source to video in pano mode
preRoll string <optional>

url to pre-roll ad (VAST advertisement source)
midRolls Array.<string> <optional>

arrays of mid-roll ads urls (VAST advertisement sources)
mode hd | pano <optional>
 hd

player active video mode (hd by default)
Example
const hdSource = 'https://some.domain.com/path/to/video/hd/video.mp4';
const panoSource = 'https://some.domain.com/path/to/video/pano/video.mp4';
const preRoll = 'https://ads/preroll/url';
const midRolls = ['https://ads/midroll/url1', 'https://ads/midroll/url2'];

// Set single source, by default handled as HD
player.setSource(hdSource);

// Set single pano source
player.setSource(panoSource, null, null, 'pano'); // will attach source to the pano mode and start player in pano mode
// player.setSource(panoSource); // in case if you want to play pano in fullstatic hd mode, without ability to pan/zoom.

// Set both pano|hd sources, also UI icon for switch sources will be shown on player UI
player.setSource({ hd: hdSource, pano: panoSource });

// Set source with Ads
// player.setSource(hdSource, preRoll, midRolls);
