Migration from 1.x version

There are few major changes which we had during migration from 1x to 2x version of WebSDK.

  • Including script

In 1.x version script looked like

<script src="https://pixellot-web-sdk.pixellot.tv/{projectID}/v-{version}/pixellot-web-sdk.js"></script>

In 2.x version we removed projectID from url, so it became more clear.. but instead if you want to use SDKApi and Analytics you need to set your projectID manually.

<script src="https://pixellot-web-sdk.pixellot.tv/v-{version}/pixellot-web-sdk.js"></script>

const WebSDK = window['pixellot-web-sdk'];
WebSDK.SDKApi.auth.setProjectID('00x00xxxx000xx00x0');

  • Decorators deprecated - Plugins introduction

We started to deprecate decorators practice as it's hard to manage it and code becomes more dirty with it.

Now instead of decorators you can use plugins. If you still need for some reasons it we still provide it under Decorators option as before.

!!! Be aware that in future releases we will remove Decorators completely, so please move to Plugins instead.

Following examples for comparison:

  • Decorators setup:
const { Player: PixellotPlayer } = window['pixellot-web-sdk'].Player;
const { TimeMoveDecorator, TagCreateDecorator } = window['pixellot-web-sdk'].Decorators;

const player = new PixellotPlayer('<div id="player"></div>');
const timeMoveDecoratedPlayer = TimeMoveDecorator(player)
const tagCreateDecoratedPlayer = TagCreateDecorator(timeMoveDecoratedPlayer, { /* options */ })

// Then you need to use `tagCreateDecoratedPlayer` as it's last one decorated...

const myPlugins = [
    TimeMoveDecorator,
    [TagCreateDecorator, { /* options */ }]
];

player.registerPlugins(myPlugins);
  • Plugins setup
const { TimeMovePlugin, TagCreatePlugin } = window['pixellot-web-sdk'].Plugins;
const myPlugins = [TimeMovePlugin, [TagCreatePlugin, { /* options */ }]];

player.registerPlugins(myPlugins);
// That's it!

  • SDK Provided services - now SDKApi!

Previously we provided different services for communication with Tags/Clits/etc under different properties. Now it's all in one.

OLD usage

const { TagService, ClipService } = window['pixellot-web-sdk'];

ClipService.getClips({ limit: 10, offset: 0 }); // 
TagService.getTags({ limit: 10, offset: 0, videoType: 'live' }); //

NEW usage (with SDKApi)

const { SDKApi } = window['pixellot-web-sdk'];

SDKApi.getClips({ limit: 10, offset: 0 }); // 
SDKApi.getTags({ limit: 10, offset: 0, videoType: 'live' }); //

!!! BTW you still can use the old option, but we're also deprecating it, so please use SDKApi prop instead.
